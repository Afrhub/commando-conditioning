# Commando Conditioning — Design Context

## Users

**Who:** 18-32 year old men preparing for Royal Marines entrance test. Pre-enlistment candidates. Some current-or-ex military, mostly civilian. Physically motivated, methodical, used to following protocols.

**Context of use:**
- Dawn or evening — pre-training or post-training entry. Often gym/track-side on phone.
- 12-week commitment, opened daily. Familiar surface after week 1.
- High intent — they self-selected for this. Not casual fitness browsers.
- Single-user app. No social loop, no leaderboard, no streaks-as-gamification.

**Job to be done:** Show me today's session. Track what I completed. Confirm I'm on track for the test. Don't waste my time.

## Brand Personality

**Three words:** disciplined · precise · unsentimental

**Voice:**
- Terse, military-clinical. No exclamation marks. No emojis. No marketing fluff.
- Numbers over adjectives. "Press-ups: 1 set to failure" not "Crush those press-ups!"
- Acknowledges the standard, never lowers it.

**Emotional goal:** confidence through clarity. The user should feel the app respects their seriousness — not coddled, not hyped, just shown the work.

## Aesthetic Direction

**Primary reference:** Apple Fitness+ / Apple Watch UI. Big monochrome tiles, generous whitespace, soft shadow ramps, single accent color used sparingly. Editorial typography for headings, system stack for body.

**Layered with:** military precision. Bold uppercase metadata labels. Tabular numerals everywhere figures appear. Rule-of-three structure (set 1 / set 2 / set 3 always equal-width tiles, not a list).

**Anti-references — explicitly DO NOT look like:**
- Generic fitness apps (Fitbit, Apple Health pastels, ring-chart obsession, friendly-coach tone)
- Hardcore tactical brands (camo, skull/eagle icons, blood-red, "operator" aesthetic)
- Crypto/SaaS dashboards (purple-blue gradients, glassmorphism, neon accents, sparkline decoration)
- Garmin/Strava data-density (athlete-as-spreadsheet)

**Theme:** dark only. `--color-bg` = oklch tinted neutral near `#0a0a0a`. Premium-clinical reads better on dark; brand mark is monochrome and inverts cleanly.

**Color discipline:**
- 60% surface (tinted near-black neutrals)
- 30% text + borders (warm neutrals, never gray-on-gray)
- 10% accent — split: rm-green for completed/success, accent-gold (`#d4a017`) for test-day markers and primary CTAs only

The accent budget is strict. If everything is gold, nothing is.

## Design Principles

1. **Tile, don't list.** Every repeated unit (sets, days, weeks) is a tile of equal weight. Never bullet points where a row of tiles would do.
2. **Numbers are typography.** Reps, distances, times, levels — render in oversized tabular display weight. The number IS the headline.
3. **One accent, used rarely.** Gold = "this is important right now" (CTA, test day). Green = "this is done." Everything else neutral.
4. **Restrained motion.** Transitions are short (180-240ms), exponential ease-out, single transforms. No bounce, no elastic. Scroll-driven parallax in selective places (week grid, hero on schedule open). Otherwise stillness.
5. **Generous breathing room.** Padding scales up at larger viewports. The page never feels packed. Empty space is part of the design, not a missing element.

## Component Primitives

| Primitive | Use |
|---|---|
| `<Surface>` | Base card. radius-md, surface bg, shadow-1, padding-5. The atom. |
| `<Tile>` | Square or fixed-ratio interactive surface. radius-lg, surface bg, shadow-1 → shadow-2 on hover, scale 1.0 → 1.01. Used for day cells, set tiles, week summaries. |
| `<Stat>` | Display a number with label below. Number in display font, tabular, oversized. Label uppercase, text-mute, letter-spacing. |
| `<Button>` | Three variants: primary (accent gold bg, dark text), secondary (border-only), ghost (no border, hover bg). Pill radius. No drop shadow on buttons. |
| `<Field>` | Label-input pair. Label uppercase mute. Input borderless on surface-hi background. Active state: border accent-gold 1px ring. |
| `<Header>` | Sticky top, surface-hi blur, hairline border bottom. |

## Constraints for downstream passes

**gpt-taste pass:** GSAP parallax allowed on schedule scroll (week-row stagger fade-in, hero figure pin-scrub on session open). DO NOT add parallax to set tiles or daily session lists — those need to feel instant, not theatrical. Bento grid pattern for week overview is welcome.

**emil pass:** focus on micro-states. Hover lift on tiles (translateY -2px, shadow up one step). Set-tile completion: scale-pop 1.0→1.05→1.0 over 220ms, color crossfade to rm-green. Page transitions via CSS view-transitions where supported.

**high-end pass:** type scale is locked. Don't shrink. Make the hero figures (current level, today's totals) the loudest element on screen.

**animate pass:** restrain. Single motion per state change. No simultaneous slide + fade + scale.

**polish pass:** verify shadow ramps (1→2→3 are perceptibly different), check tabular numerals everywhere figures appear, audit text-mute usage (don't fade body copy).

**audit pass:** WCAG AA contrast minimum (4.5:1 body, 3:1 large). Reduced motion = no parallax. Touch targets 44×44 minimum. Tabular numerals don't break with VoiceOver.
